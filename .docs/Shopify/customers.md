# Shopify Customer Source Schema

**Service:** `src/graphql/shopify/customers.js`  
**API version:** `2026-04` — all fields below confirmed compatible. See [api-version.md](api-version.md).  
Source data fetched via the Shopify Admin GraphQL API (`/admin/api/2026-04/graphql.json`).

---

## Customer Fields

| Field | GraphQL Type | Always Populated | Example | Notes |
|---|---|---|---|---|
| `id` | `ID` | Yes | `gid://shopify/Customer/12345` | Global Shopify GID — not forwarded to BC |
| `firstName` | `String` | No | `"Jane"` | Can be null for partial accounts |
| `lastName` | `String` | No | `"Doe"` | Can be null |
| `email` | `String` | Yes | `"jane@example.com"` | Unique per store |
| `phone` | `String` | No | `"+15551234567"` | E.164 format; can be null |
| `note` | `String` | No | `"VIP customer"` | Internal note — maps to BC `notes` |
| `state` | `CustomerState` | Yes | `ENABLED` | `ENABLED`, `DISABLED`, `INVITED`, `DECLINED` |
| `taxExempt` | `Boolean` | Yes | `false` | Native Shopify tax exemption flag — does **not** carry AvaTax code |
| `tags` | `[String]` | Yes | `["VIP", "wholesale"]` | Array; not used for customer group mapping in this project |

> **Company name is address-level.** There is no top-level `company` field on the Shopify customer. The company lives on each address object. Extract from `defaultAddress.company` for the BC `company` field.

## Marketing Consent

Shopify tracks marketing consent per channel independently.

```graphql
emailMarketingConsent {
  marketingState       # Enum — see table below
  marketingOptInLevel  # SINGLE_OPT_IN | CONFIRMED_OPT_IN | UNKNOWN
  consentUpdatedAt     # DateTime
}
```

| `marketingState` Value | Meaning | Map to BC |
|---|---|---|
| `SUBSCRIBED` | Customer opted in | `true` |
| `NOT_SUBSCRIBED` | Never opted in | `false` |
| `PENDING` | Double opt-in sent, not confirmed | `false` |
| `UNSUBSCRIBED` | Explicitly opted out | `false` |
| `INVALID` | Email is invalid | `false` |
| `REDACTED` | GDPR-redacted | `false` |

## Addresses

`defaultAddress` is a single `MailingAddress` object. `addresses` is a **plain array** (not a connection — no `edges`/`node`, no `first` argument) and always includes the default address — do not create `defaultAddress` a second time when iterating `addresses`.

### MailingAddress Fields

| Field | Type | Notes |
|---|---|---|
| `id` | `ID` | Shopify address GID — not forwarded to BC |
| `firstName` | `String` | May differ from customer `firstName` |
| `lastName` | `String` | May differ from customer `lastName` |
| `company` | `String` | The customer's company name lives here |
| `address1` | `String` | Street line 1 |
| `address2` | `String` | Unit/suite/apt — can be null |
| `city` | `String` | City name |
| `province` | `String` | Full name (e.g. `"Texas"`) — use `provinceCode` for BC |
| `provinceCode` | `String` | ISO 3166-2 subdivision (e.g. `"TX"`) |
| `country` | `String` | Full name (e.g. `"United States"`) — use `countryCodeV2` for BC |
| `countryCodeV2` | `String` | ISO 3166-1 alpha-2 (e.g. `"US"`) |
| `zip` | `String` | Postal code |
| `phone` | `String` | Address-level phone — may differ from customer `phone` |

## Store Credit

Store credit is **Shopify Plus only**. Standard Shopify stores do not expose this via the Admin GraphQL API — querying `storeCreditAccounts` on a non-Plus store will return an error or empty connection.

```graphql
storeCreditAccounts(first: 5) {
  edges {
    node {
      balance {
        amount       # String (e.g. "25.00")
        currencyCode # "USD"
      }
    }
  }
}
```

**Migration rule:** Sum all account balances for the customer. Write to BC `store_credit_amounts: [{ amount: <total> }]`. If not available, omit the field (defaults to 0).

## Metafields

Fetched via aliased singular `metafield(namespace, key)` — same pattern as products. See [metafields.md](metafields.md) for why this approach is used over `metafields(identifiers: [...])`.

| Namespace | Key | Alias | Notes |
|---|---|---|---|
| `avatax_excise` | `customer_type` | `mf_avatax_customer_type` | AvaTax customer type classification |
| `avatax_excise` | `customer_no` | `mf_avatax_customer_no` | AvaTax excise tax exempt number |
| `adv_reg` | `EIN-Field` | `mf_adv_reg_ein_field` | EIN / federal tax ID. Key contains a hyphen — alias uses underscores |
| `limits` | `exempt_order_limits` | `mf_limits_exempt_order_limits` | Order limit exemption configuration |
| `configuration` | `disable_cart_buttons` | `mf_configuration_disable_cart_buttons` | Per-customer cart button visibility flag |
| `custom` | `purchasing_list_subscription` | `mf_custom_purchasing_list_subscription` | Purchasing list subscription status |

All 6 are fetched as aliased singular `metafield(namespace, key)` calls. Only non-null values are written to BC. To add more, append to `CUSTOMER_METAFIELD_DEFS` in `src/graphql/shopify/customers.js`.

## Source Data Caveats

- **Company is address-level.** Extract from `defaultAddress.company`. If the customer has no addresses, `company` will be blank in BC.
- **Phone is E.164.** Shopify stores phone in international format (`+15551234567`). BC accepts this format natively.
- **`storeCreditAccounts` requires Shopify Plus.** Wrap in a try/catch or conditional query.
- **`defaultAddress` is included in `addresses`.** Iterating `addresses` to build the BC address list will include the default — identify it by matching ID to `defaultAddress.id`, create it first, then create the rest.
- **Duplicate addresses are common.** Shopify allows saving near-identical addresses with minor formatting variations. Normalize and deduplicate before writing to BC. See [customer-data-shape.md](../BigCommerce/customer-data-shape.md#address-deduplication).
- **`addresses` is a plain array, not a connection.** No `edges`/`node` wrapper, no `first` argument. Returns all addresses for the customer with no pagination — confirmed via runtime error against the 2026-04 API.
