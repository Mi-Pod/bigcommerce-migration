# Shopify Pages — API Reference (2026-04)

Pages are static content pages in the Shopify Online Store (e.g. "About Us", "Contact"). They are distinct from blog posts and product descriptions.

---

## Key Concepts

- `isPublished` — boolean; controls storefront visibility
- `body` — HTML string; the page's main content
- `templateSuffix` — optional Liquid template variant (e.g. `"contact"` → uses `page.contact.liquid`); null means default `page.liquid`
- `publishedAt` — ISO 8601 string; can be in the future for scheduled publish
- `author` — **does not exist** on `Page` type in Admin GraphQL; omit from queries
- `bodySummary` — **does not exist** on `Page` type in Admin GraphQL; omit from queries

---

## Count Pages

```graphql
query PagesCount {
  pagesCount { count }
}
```

---

## Paginate Pages

```graphql
query GetPagesPage($first: Int!, $after: String) {
  pages(first: $first, after: $after) {
    pageInfo { hasNextPage endCursor }
    edges {
      node {
        id
        title
        handle
        isPublished
        publishedAt
        createdAt
        updatedAt
        body
        templateSuffix
      }
    }
  }
}
```

Max page size: **250**

---

## Fetch Single Page by GID

```graphql
query GetPage($id: ID!) {
  page(id: $id) {
    id
    title
    handle
    isPublished
    publishedAt
    createdAt
    updatedAt
    body
    templateSuffix
  }
}
```

GID format: `gid://shopify/OnlineStorePage/123456789`

---

## Export Strategy

1. Call `pagesCount` for total count.
2. Paginate with `pages(first: 250, after:)` until exhausted.
3. Save each page as `exports/content/pages/data/{handle}.json`.
4. Append a row to `exports/content/pages/index.csv`.

---

## Notes

- The GID type is `OnlineStorePage` (not just `Page`) — confirm this when constructing GIDs manually.
- `body` is HTML; preserve verbatim for BigCommerce migration.
- `templateSuffix` is useful for identifying pages that have custom layouts (e.g. landing pages) — these may need extra attention during migration.
- `author` is a plain string, not a user reference; it stores the name at time of creation and does not update if the user's name changes.
- Pages do not support tags, unlike blog articles.
- `bodySummary` is auto-generated by Shopify and cannot be set directly via API; do not rely on it for migration content.
